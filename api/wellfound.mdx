---
title: "Wellfound"
description: "Fetch Wellfound company profiles or founders/team as structured JSON."
---

Two modes:

- **People** (default) — founders and team (LinkedIn often present); use `find_email`
- **Company** (`mode=company` or `find_persona`) — company profile with website domain for personas

## Endpoint

```text
POST https://app.lens.ly/api/v1/scrapers/wellfound
```

## Request parameters

| Parameter | Required | Description |
| --- | --- | --- |
| `slug` | Conditional | Company slug |
| `url` | Conditional | `wellfound.com/company/{slug}` URL |
| `mode` | No | `people` (default) or `company` |
| `find_email` / `find_linkedin` | No | People mode — email waterfall (LinkedIn often already on row) |
| `find_persona` | No | Company mode — personas at company domain |
| `persona_titles` | With `find_persona` | Job titles (max 3) |
| `webhook_url` / `delivery` | No | Async delivery options |

## Example — people \+ email

```bash
curl -X POST https://app.lens.ly/api/v1/scrapers/wellfound \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"slug":"airbnb","find_email":true}'
```

## Example — company \+ persona

```bash
curl -X POST https://app.lens.ly/api/v1/scrapers/wellfound \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "slug": "airbnb",
    "mode": "company",
    "find_persona": true,
    "persona_titles": ["CEO", "CTO"]
  }'
```
