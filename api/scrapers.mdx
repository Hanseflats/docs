Scrapers are a separate product surface from [Audiences / competitor-reviews](/api/competitor-reviews). They rebuild review and startup pages via Scrapingdog (Netrows for G2), always return **structured JSON** (never raw markdown), and support optional LinkedIn / persona / email enrichments.

## Endpoint

```text
POST https://app.lens.ly/api/v1/scrapers/{slug}
```

Valid `slug` values: `trustradius`, `trustpilot`, `glassdoor`, `yc`, `trustmrr`, `g2`.

## Common body

| Parameter | Required | Description |
| --- | --- | --- |
| `url` | Usually | Website **or** platform URL. Website → SERP discover. Optional for YC search filters. |
| `page` | No | Start page (default `1`) |
| `max_pages` | No | Pages to scrape (default `1`). `>1` forces async. |
| `keyword` | No | Glassdoor keyword (KH) pain page |
| `query` / `batch` / `industry` / … | No | YC Algolia search filters (`batch` accepts `W24` → `Winter 2024`) |
| `find_linkedin` | No | Resolve reviewer LinkedIn (\+1 credit per success) |
| `find_persona` | No | Company scrapes only; requires `persona_titles` (max 3) |
| `find_email` | No | Email enrichment (see below) |
| `email_provider` | No | `findymail` or `leadmagic` for BYO sync path |
| `email_api_key` | No | Request override (not persisted) |
| `webhook_url` | No | Delivery webhook |
| `delivery` | No | `chunks` (default), `item`, or `complete_only` |
| `webhook_batch_size` | No | Default `50`, max `100` |

## Sync vs async

- **Sync** — `max_pages === 1`, no `find_persona`, and either no `find_email` or BYO email key. Response: `{ data, credits_used, meta }`.
- **Async** — multi-page, `find_persona`, or RB bulk `find_email`. Response `202`: `{ job_id, status, poll }`.

Poll: `GET /api/v1/scrapers/jobs/{id}`<br />Results: `GET /api/v1/scrapers/jobs/{id}/results?cursor=&limit=100`

## Email enrichment

| Mode | How | Credits |
| --- | --- | --- |
| Sync / single | BYO Findymail or LeadMagic key | 0 email credits |
| Async bulk | Our RevenueBase (LinkedIn → name\+domain) | \+1 per email found |

Save BYO keys:

```text
PUT /api/v1/integrations/email-finders
{ "findymail_api_key": "…" }
```

## Credits (scrape base)

| Scraper | Per page |
| --- | --- |
| TrustRadius / Trustpilot / TrustMRR / YC | 1 |
| Glassdoor / G2 | 2 |

Website discovery is included. `find_linkedin` / `find_persona` / RB email add credits as above.

## Example (sync)

```bash
curl -X POST https://app.lens.ly/api/v1/scrapers/trustradius \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://www.trustradius.com/products/hubspot-crm/reviews",
    "max_pages": 1
  }'
```

## Example (async \+ webhook chunks)

```bash
curl -X POST https://app.lens.ly/api/v1/scrapers/trustpilot \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "hubspot.com",
    "max_pages": 3,
    "find_linkedin": true,
    "find_email": true,
    "webhook_url": "https://hooks.example.com/scraper",
    "delivery": "chunks"
  }'
```

Webhook events: `scraper.records` (batched) then `scraper.completed` with `results_url`. Full results always live on the job — webhook is a delivery stream, not the source of truth.

## Glassdoor

Pass a company website or Glassdoor URL. Optional `keyword` builds a KH reviews URL from the resolved `{ companySlug, employerId }` handle.

## YC

- Search: omit company URL and pass `query` / `batch` / filters (`mode: "search"`).
- Company: `url` = `https://www.ycombinator.com/companies/{slug}` or a website for discovery. Use `find_persona` \+ `persona_titles` for decision-makers.
