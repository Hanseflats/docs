title: "Webhooks"
description: "Audience destination webhooks (one POST per row) and legacy job completion webhooks."
---

Lens supports three webhook styles: **audience destinations**, **scraper job delivery**, and **legacy job completion**.

## Audience destinations

Configure `delivery_webhook_url` on [Audiences](/api/audiences) launch or `PATCH`. When new rows are ready (pipeline auto-sync or `POST /api/v1/audiences/:id/sync`), Lens POSTs **once per lead or company**.

```json
{
  "audience_id": "...",
  "play_slug": "competitor-reviews",
  "title": "HubSpot reviewer leads",
  "type": "leads",
  "lead": {
    "email": "alex@acme.com",
    "first_name": "Alex",
    "last_name": "Lee",
    "job_title": "VP Marketing",
    "company_domain": "acme.com",
    "created_at": "2026-07-18T12:00:00.000Z"
  },
  "synced_at": "2026-07-18T12:01:00.000Z"
}
```

Company-only audiences send `"type": "companies"` and a `"company"` object.

<Info>
  Use HTTPS. Return **2xx** within 30 seconds. Concurrency is capped (about 5 parallel POSTs). Instantly destinations stay bulk and are separate from webhooks.
</Info>

## Scraper webhooks

Pass `webhook_url` on `POST /api/v1/scrapers/{slug}` (async jobs). Optional `delivery`:

| `delivery` | Behavior |
| --- | --- |
| `chunks` (default) | Batches of records (`webhook_batch_size`, default 50, max 100), then a completion event |
| `item` | One POST per record (auto-downgrades to chunks above 100 records), then completion |
| `complete_only` | Only the completion event (fetch records via results URL) |

**Requirements:** HTTPS only. No private/link-local hosts. Redirects are not followed. Return **2xx** within 30 seconds. We retry failed deliveries a few times.

### Chunk / item payloads

```json
{
  "event": "scraper.records",
  "job_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "slug": "trustradius",
  "batch_index": 0,
  "batch_total": 2,
  "is_final_batch": false,
  "records": [{ "reviewer_name": "Alex L.", "rating": 8 }]
}
```

Per-record mode uses `event: "scraper.record"` with `index` and `record` instead of `records`.

### Completion payload

```json
{
  "event": "scraper.completed",
  "job_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "status": "completed",
  "record_count": 40,
  "credits_used": 3,
  "results_url": "https://app.lens.ly/api/v1/scrapers/jobs/a1b2c3d4-e5f6-7890-abcd-ef1234567890/results"
}
```

On failure, `status` is `"failed"` and `error` may be present. See [Scrapers overview](/api/scrapers).

## Legacy job webhooks

Some older job APIs still accept `webhook_url` on create. On completion we POST a single job payload:

```json
{
  "job_id": "job_abc123",
  "status": "completed",
  "results": [],
  "summary": { "totalReviews": 50, "profilesFound": 42 }
}
```

That path is for legacy jobs, not Playbooks audience sync or the scrapers API.
